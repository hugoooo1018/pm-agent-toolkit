# 字段命名规范

全文字段名应遵循以下约定，保证 PRD / 业务对象 / 原型图 / mermaid 节点四处说法一致。

## 基本原则

- **业务语境优先**：字段显示名用中文业务术语，不暴露数据库列名或驼峰命名
- **全文加粗**：核心业务字段每次出现都加粗（`**字段名**`），加粗文本每次必须完全一致
- **中英混排留空格**：中文 + 英文缩写之间留半角空格

## 强制命名

### ID 类字段

中文词 + 半角空格 + `ID`

| 正例 | 反例 |
|---|---|
| `规则 ID` | ❌ `规则ID`（无空格） |
| `策略 ID` | ❌ `rule_id`（暴露技术命名） |
| `广告账户 ID` | ❌ `ad_account_id` |

### URL / 链接类字段

中文词 + 半角空格 + `URL`

| 正例 | 反例 |
|---|---|
| `规则 URL` | ❌ `规则URL` / `rule_url` |
| `登录 URL` | ❌ `登录链接`（不统一） |

### 时间字段

中文双字命名，不暴露 `_at` 后缀

| 正例 | 反例 |
|---|---|
| **创建时间** | ❌ `created_at` / `创建于` |
| **更新时间** | ❌ `updated_at` / `最后更新` |
| **执行时间** | ❌ `executed_at` |

### 状态字段

中文 + 枚举值用双引号包裹

```markdown
| **规则状态** | 是 | 枚举：`"生效中"` / `"未生效"` | "请选择规则状态" |
```

反例：
- ❌ `status: active / inactive`（暴露枚举原值）
- ❌ 规则状态：已启用（不一致）

### 操作按钮 / UI 标签

PRD 中出现的按钮名用**英文原文**（与真实 UI 一致）+ 双引号包裹，例：

- `"Create Rule"` / `"Edit"` / `"Delete"` / `"Run"` / `"Back to Rules"`

反例：
- ❌ "创建规则"按钮（与实际 UI 不符）
- ❌ Create Rule（无引号，混同正文）

### 页面名称

中文业务词 + "页"后缀；加粗

- **Rule 列表页** / **Rule 详情页** / **Strategy 详情页**

反例：
- ❌ `Rule List Page` / `规则列表` / `rule-list-page`

### 业务对象（实体）

首字母大写的英文词（与代码侧实体名一致）；加粗

- **Rule** / **Strategy** / **Ad Account** / **Campaign** / **Ad Set** / **Ad** / **Target Object**

反例：
- ❌ "rule" / "Rules"（复数不统一） / "广告账户"（某些场景改成中文）

### 术语复合名（加粗不可拆）

- **规则状态 Toggle 开关** —— 整体作为一个术语，每次出现必须完整引用
- **执行顺序** / **作用层级** / **命中即停**

反例：
- ❌ 状态 Toggle / 规则 toggle / 状态开关（拆开或改写）

## 字段表中的写法

字段表第一列字段名必须加粗：

```markdown
| 字段 | 必填 | 校验规则 | 校验失败提示 |
| --- | --- | --- | --- |
| **规则名称** | 是 | 不为空；同一 **广告主** 下不允许重复 | "此字段为必填项" / "该规则名称已存在" |
| **Ad Account** | 是 | 单选 | "请选择一个 Ad Account" |
```

校验规则列里引用其他业务对象时也要加粗。

## 业务对象文档中的写法

`{module}-business-objects.md` 里字段表固定三列：

```markdown
| 属性 | 说明 | 是否由用户配置 |
| --- | --- | --- |
| **规则名称** | 用户自定义的 Rule 显示名称 | 是 |
| **规则 ID** | 唯一标识 | 自动生成 |
| **创建时间** | 系统自动生成 | 自动生成 |
```

第三列用"是 / 否 / 自动生成"三种枚举，不用其他措辞。

## 一致性自检

写完 PRD 后必须跑一遍：

- 同一字段在字段表、正文、mermaid 节点、原型图、业务对象文档中**文本完全一致**
- 同一字段在四处的顺序完全一致（功能清单 / 架构图 / 功能详述 / 原型图）
- 新增字段时同步更新业务对象文档
