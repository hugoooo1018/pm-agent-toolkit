# 形态 A：操作型功能模板

适用：创建 / 编辑 / 删除 / 状态切换 / 搜索 等"写操作"或"动作操作"。

## 模板

```markdown
**{功能名称}**

- 触发方式：{入口 + 交互形式，如"在 X 页面点击 Y 按钮，打开弹窗"}
  - 前置条件：{执行前必须满足的业务状态；可选子 bullet}

**弹窗表单字段：**（加粗小标题，仅当存在表单时；非表单型功能删去此节）

| 字段 | 必填 | 校验规则 | 校验失败提示 |
| --- | --- | --- | --- |
| **{字段名}** | 是 / 否 | {静态校验；业务校验（标注作用域）} | "{中文提示}" |

- 查询规则：{搜索功能专用；定义匹配规则、重置规则；非搜索功能删去此行}
- 校验时机：{详见业务规则 **表单校验时机** 或单独说明}
- 提交结果状态：
  - 取消：{仅二次确认型；弹窗关闭，数据不变}
  - 成功：{弹窗关闭 + 列表刷新 + Toast；系统自动生成的 ID / 创建时间 / 默认状态一并写入此处}
  - 失败：{弹窗保持打开 + 字段下错误提示}
- 约束与边界：（可选；仅当确有业务边界或数据影响时列出；两类都没有则整段删除）
  - 业务边界：{本功能不做什么 / MVP 仅覆盖哪些维度}
  - 数据影响：{破坏性后果：不可恢复 / 级联删除 / 不回滚外部 API 副作用}
- 交互规则：{可选平级 bullet；UI 约定，如"状态切换无需二次确认"}
- 展示规则：{可选平级 bullet；排序 / 默认值等}
```

## 要素说明

### 「触发方式」
必含入口 + 交互形式，不能只写动作。入口包含两要素："在 **X 页面**" + "点击 **Y 按钮**"（或输入、勾选等）。

### 「前置条件」
仅当存在业务状态门禁时写（如"仅未生效可删除"）。放在触发方式下的子 bullet，**不得**上提到约束与边界。

### 「弹窗表单字段」表
- 表头固定 4 列：字段 / 必填 / 校验规则 / 校验失败提示
- 字段名全文加粗
- 校验规则多条用中文分号（；）分隔
- 业务校验必须标注作用域（例："同一 **广告主** 下不允许重复"）
- 校验失败提示用中文直引号包裹：`"..."`；多种失败场景用 ` / ` 分隔多个提示

### 「提交结果状态」三件套
- **取消**：仅二次确认型操作才写；常规表单提交不含取消分支
- **成功**：所有自动生成字段（ID / 创建时间 / 默认状态）集中写在此处，不单列「系统自动处理」
- **失败**：明确错误展示位置（字段下方 / 顶部 / Toast）

### 「约束与边界」严格两类
- 只装 **业务边界** 与 **数据影响**
- 无则整段删除；严禁写 "业务边界：无" / "数据影响：无"

## 正例（摘自 `rule-management-prd-v2.md`）

### 正例 1：带表单的创建操作

```markdown
**创建 Rule**

- 触发方式：在 **Rule 列表页** 点击 "Create Rule" 按钮，打开 Rule 创建弹窗

**弹窗表单字段：**

| 字段 | 必填 | 校验规则 | 校验失败提示 |
| --- | --- | --- | --- |
| **规则名称** | 是 | 不为空；同一 **广告主** 下不允许与其他 **Rule** 的 **规则名称** 重复 | "此字段为必填项" / "该规则名称已存在" |
| **Ad Account** | 是 | 单选，一条 **Rule** 只能绑定一个 **Ad Account** | "请选择一个 Ad Account" |
| **作用层级** | 是 | 从 **Campaign** / **Ad Set** / **Ad** 中单选；默认选中 **Campaign** | "请选择作用层级" |

- 校验时机：详见业务规则 **表单校验时机**
- 提交结果状态：
  - 成功：弹窗关闭，列表在当前搜索下刷新；Toast "Rule 创建成功"；系统生成 **规则 ID** 和 **创建时间**；**规则状态** 默认"未生效"
  - 失败：弹窗保持打开，在对应字段下方显示错误提示
```

要点注解：
- 表单外无「约束与边界」段落 → 因为此功能既无业务边界也无数据影响，整段删除
- 自动生成的 ID / 时间 / 默认状态集中写在「成功」分支
- 无「取消」分支 → 因为非二次确认型

### 正例 2：带前置条件 + 约束与边界的删除操作

```markdown
**删除 Rule**

- 触发方式：在 **Rule 列表页** 或 **Rule 详情页** 点击 "Delete" 按钮，弹出二次确认弹窗
  - 前置条件：仅 **规则状态** 为"未生效"的 **Rule** 可删除；"生效中"的 **Rule** "Delete" 按钮置灰不可点击
- 确认弹窗：告知用户删除不可恢复且该 **Rule** 下全部 **Strategy** 将一并删除，提供确认与取消两个操作
- 提交结果状态：
  - 成功（取消）：弹窗关闭，**Rule** 及其 **Strategy** 保持不变，列表保持当前状态
  - 成功（确认删除）：**Rule** 及其全部 **Strategy** 从系统中移除，列表刷新；Toast 提示 "Rule 已删除"
  - 失败（确认删除）：弹窗保持打开，页面顶部显示错误提示
- 约束与边界:
  - 数据影响：删除后无法恢复，连带删除其下全部 **Strategy**；仅清除本系统侧配置，不回滚已通过 Meta Marketing API 执行的 **动作** 副作用
```

要点注解：
- 前置条件在触发方式下（非在约束与边界里）
- 约束与边界只含 **数据影响** 一类 → 没有业务边界就不写业务边界
- 二次确认型有 3 个 "成功" 分支（取消 / 确认删除成功 / 确认删除失败）

### 正例 3：搜索 / 查询型

```markdown
**Rule 列表查询**

- 触发方式：在 **Rule 列表页** 关键词搜索框输入关键词，按 Enter 键或点击查询按钮触发
- 查询规则：
  - 匹配规则：Query 作为连续子串出现在 **规则名称** 中即视为命中；大小写不敏感；Query 首尾空格自动忽略，中间空格原样保留参与匹配
  - 重置规则：手动清空搜索框后按 Enter 键或点击查询按钮，恢复完整 **Rule** 列表
- 提交结果状态：
  - 成功（命中）：**Rule** 列表刷新为匹配结果
  - 成功（未命中）：**Rule** 列表展示空状态文案
  - 失败：页面顶部显示错误提示
- 约束与边界：
  - 业务边界：仅支持按 **规则名称** 单一维度匹配，不支持跨字段搜索
- 展示规则：搜索结果继承 **Rule 列表页** **创建时间升序** 排列
```

要点注解：
- 搜索型用「查询规则」代替「弹窗表单字段」
- 约束与边界只含 **业务边界** 一类
- 「展示规则」是平级 bullet，不入约束与边界

## 反模式（严禁出现）

- ❌ 创建与编辑合并写成一段
- ❌ 字段表少于 4 列或列序错乱
- ❌ 把前置条件写进约束与边界
- ❌ 自动生成的 ID / 时间单独写「系统自动处理」
- ❌ 字段校验只写规则不给中文提示文案
