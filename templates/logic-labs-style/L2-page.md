# 形态 B：页面型功能模板

适用：列表页 / 详情页 / 配置页等"承载多个操作入口"的页面。

## 模板

```markdown
**{页面名称}**

- 入口：{从哪里进入；如"用户通过侧边栏点击 X 进入"}
- 页面布局：
  - 顶部区域：{展示内容 + 可执行操作；多个操作按从左到右或从上到下的顺序列出}
  - 主体区域：{列表 / 表格 / 表单；列出字段与行内操作}
  - 空状态：{标题 + 描述文案；英文原文直引}
- 展示规则：{排序规则、分页规则；可选}
```

## 要素说明

### 「入口」
必须是可定位的操作路径，不能只写"用户进入"。例："**广告主** 通过侧边栏点击 Rule 进入"。

### 「页面布局」
- 按视觉分区列出（顶部 / 主体 / 底部 / 侧栏）
- 每个分区写明展示内容 + 可执行操作
- **每个操作必须"详见 XXX"引用到对应的 L2 操作型功能**，不重复展开操作细节

### 主体列表 / 表格的列定义
- 逐列列出，每列说明内容与行内操作
- 可点击跳转的字段 → 用"详见 XXX"引用目标页

### 「空状态」（硬红线）
- 必须有，缺失即 Never do 命中
- 标题 + 描述两部分都要写
- 描述用英文原文直引（避免前端翻译后丢失语义）
- 例：`No rules yet. Click "Create Rule" to create your first rule.`

### 「展示规则」
- 排序规则：明确按哪个字段、升 / 降序、是否可手动调整
- 分页规则：MVP 一次性加载或分页加载、页大小

### 多个 subsection
- 页面内若有独立子区域（如详情页里的"策略列表"区），可单独写"{子区域名} 展示规则"小节

## 正例 1：列表页

```markdown
**Rule 列表页**

- 入口：**广告主** 通过侧边栏点击 Rule 进入
- 页面布局：
  - 顶部区域：
    - 左侧：关键词搜索框，详见「Rule 列表查询」
    - 右侧："Create Rule" 按钮，详见「创建 Rule」
  - 主体区域：**Rule** 列表表格，列为：
    - 序号
    - 规则名称，详见「Rule 详情页」
    - 规则状态 Toggle 开关，详见「Rule 状态开关」
    - 作用层级
    - Ad Account
    - 操作："Run" 按钮，详见「Rule 手动执行」；"Delete" 按钮，详见「删除 Rule」
  - 空状态：**Rule** 列表为空时展示空状态文案
- 展示规则：
  - 排序规则：**Rule** 按 **创建时间升序** 排列，不可手动调整顺序
  - 分页规则：MVP 阶段一次性加载全部 **Rule**，不分页
```

要点注解：
- 顶部区域的搜索框与创建按钮都"详见 XXX"引用对应 L2 操作型功能
- 主体区域每个操作都引用到对应功能点，不在此展开细节
- 空状态段落存在（即使具体文案在原型图里，此处也必须提及）
- 展示规则是独立平级 bullet，不写入约束与边界

## 正例 2：详情页（含子区域）

```markdown
**Rule 详情页**

- 入口：在 **Rule 列表页** 点击某条 **Rule** 的 **规则名称**
- 页面布局：
  - 顶部区域：
    - 页面标题：**规则名称**
    - 左上角："Back to Rules" 返回按钮
    - 操作栏（依次）：
      - 规则状态 Toggle 开关，详见「Rule 状态开关」
      - "Run" 按钮，详见「Rule 手动执行」
      - "Edit" 按钮，详见「编辑 Rule 属性」
      - "Delete" 按钮，详见「删除 Rule」
  - Rule 属性区域：**作用层级** / **Ad Account** / **创建时间** / **更新时间**
  - 主体区域：**策略列表** 表格，右侧 "Create Strategy" 按钮，详见「创建 Strategy」；列为：
    - 执行顺序
    - 策略名称，详见「Strategy 详情页」
    - 操作："Delete" 按钮详见「删除 Strategy」
  - 空状态：**策略列表** 为空时展示空状态文案
- **策略列表** 展示规则：
  - 排序规则：**Strategy** 按 **执行顺序** 升序排列，不可手动调整顺序
  - 分页规则：MVP 阶段一次性加载全部 **Strategy**，不分页
```

要点注解：
- 顶部操作栏严格按 UI 顺序列出（状态 → 执行 → 编辑 → 删除），与架构图 / 原型图顺序保持一致
- 属性区域只展示字段，不含操作
- 主体区域是"策略列表"子区域，所以单独写「**策略列表** 展示规则」小节
- 所有操作点都引用对应 L2 功能，不重复展开

## 反模式（严禁出现）

- ❌ 列表 / 表格缺空状态文案
- ❌ 布局描述把操作细节展开写（应该只引用对应操作型功能点）
- ❌ 同一操作在两处重复写触发方式与字段表
- ❌ 字段顺序与功能需求清单 / 架构图 / 原型图不一致
- ❌ 用中文"暂无数据"作为空状态（必须英文原文直引）
- ❌ 把排序 / 分页写进约束与边界（应放展示规则）
